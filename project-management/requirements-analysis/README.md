# Requirements Analysis

## Objective

The purpose of this phase of the project is to make any adjustments to technical and functional requirements documented in the task plan to achieve objectives under reasonable constraints as the project evolves. This may include categorizing requirements further into needs and wants, identifying requirement trade-offs through decision schematics, and differentiating between the technical complexity and importance of requirements in requirements decision matrices for future prioritization.

## Next Steps

The next milestone in the project workflow is system design and involves developing black box diagrams, use case workflows, system architectural diagrams, and UI designs.
